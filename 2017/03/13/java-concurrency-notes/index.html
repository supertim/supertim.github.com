java-concurrency-notes

1 概念

  • 同步和异步
    • 同步：方法结束才能返回
    • 异步：一个消息传递，立刻返回，在另一个线程真实执行
  • 并发和并行
    • 并发：同时执行
    • 并行：交替执行
  • 临界区
  • 阻塞和非阻塞
    • 阻塞：占用资源的线程不释放资源，其他临界区上得线程不能工作
    • 非阻塞：没有一个线程可以妨碍其他线程执行
  • 死锁,饥饿,活锁
    • 饥饿：线程无法得到资源，优先级低得线程始终得不到高优先级线程释放的资源
    • 活锁：资源不停的在两个线程间跳动

      并发级别

  • 阻塞：synchronized或者重入的
  • 无饥饿：公平锁
  • 无障碍obstruction-free：最弱的非阻塞调度，使用标记位记录是否修改，如果修改，则回退，这是一种乐观的控制，认为大多数情况下不会冲突

  • 无锁lock-free：必须有个线程有限步离开临界区

  • 无等待wait-free： 所有的线程都是有限步离开临界区，eg. ReadCopyUpdate

两个定律

  • amdahl定律
  • gustafson定律

java内存模型JMM

  • 原子性：int是，long在32位系统的读写不是原子性的
  • 可见性：线程1改掉共享变量，线程2未必能立刻知晓，原因可能是：
    • 缓存优化，硬件优化
    • 指令重排：防止流水线被打断
    • 编译器优化

  • 有序性：一个线程看起来是有序的，但是对于另一个线程未必是有序的

  • happen-before原则：指令重排的原则

    • 一个线程内保证串行性
    • volatie：变量的写，先发生于读，可见性
    • unlock必然发生于随后的lock前
    • A 先于 B， B先于C， 则A必然先于C
    • 线程的start方法由于他的每一个动作
    • 线程的所有操作先于join
    • interrupt先于被中断线程的代码
    • 构造函数执行结束先于finalize()方法
